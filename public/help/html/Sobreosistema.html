Sobre o Integrador Taxdocs Avatax

O Integrador Taxdocs × Avatax é uma aplicação corporativa, projetada com uma arquitetura moderna, modular e escalável, sendo composta por dois principais componentes:

Arquitetura da Aplicação

Backend (API)

O Backend é responsável por toda a lógica de negócio, processamento de dados, integrações externas e persistência das informações. Entre suas principais funções, destacam-se:

  • Integração com os serviços do Taxdocs (Avalara) para recuperação de documentos fiscais eletrônicos, como NFe, NFSe e CTe
  • Integração com o Avatax (Avalara) para cálculo, validação e conferência tributária
  • Processamento, normalização e armazenamento dos documentos fiscais
  • Execução de tarefas agendadas, incluindo consultas automáticas, reprocessamentos e sincronizações
  • Exposição de endpoints seguros para consumo pelo front-end
  • Registro de logs, erros e informações operacionais para auditoria, monitoramento e suporte técnico

O Backend é implementado como uma API REST, permitindo fácil integração, manutenção e evolução do sistema.

Toda a API é documentada por meio do Swagger, disponibilizando uma interface interativa que permite:

  • Visualizar todos os endpoints disponíveis
  • Consultar detalhes de requisições e respostas
  • Validar parâmetros, tipos de dados e contratos da API
  • Executar chamadas de teste diretamente pela interface
  • Facilitar a integração com sistemas externos e o suporte técnico

A documentação Swagger serve como referência oficial para desenvolvedores, equipes de integração e administradores do sistema, garantindo clareza, padronização e agilidade no uso e na evolução da API.

Front-end (Interface Web)

O Front-end é a camada responsável pela interação com o usuário, oferecendo uma interface web moderna, intuitiva e responsiva. Suas principais características incluem:

  • Visualização e acompanhamento do processamento de documentos fiscais, inclusive recuperando documentos fiscais para confronto de calculos.
  • Dashboards e indicadores operacionais
  • Consulta de documentos, status e divergências
  • Acesso a relatórios e informações consolidadas
  • Interface amigável para usuários técnicos e administrativos

O Front-end consome exclusivamente os serviços disponibilizados pelo Backend, garantindo separação de responsabilidades e maior segurança.

Modelo de Implantação On-Premise

O Integrador Taxdocs × Avatax foi projetado para utilização no modelo on-premise, ou seja, a aplicação é instalada e executada dentro da infraestrutura do próprio cliente.

Esse modelo de implantação oferece diversas vantagens, como:

  • Maior controle sobre os dados, atendendo a requisitos de segurança e compliance
  • Adequação a políticas internas de TI e governança corporativa
  • Possibilidade de integração direta com sistemas internos do cliente
  • Independência de ambientes de nuvem pública, quando necessário

Tanto o Backend quanto o Front-end podem ser hospedados nos servidores do cliente, respeitando os requisitos técnicos definidos no manual de instalação.
